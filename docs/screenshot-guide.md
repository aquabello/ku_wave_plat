# KU WAVE PLAT 완료보고서 - 화면 캡처 가이드

> 완료보고서(KU_WAVE_PLAT_완료보고서.md)의 `[화면 캡처: ...]` 위치에 삽입할 스크린샷을 촬영하기 위한 가이드.
> 각 항목별로 접속 URL, 사전 조건, 캡처 포인트를 정리한다.

---

## 기본 정보


| 항목                | URL                                                                    |
| ----------------- | ---------------------------------------------------------------------- |
| 관리 콘솔 (Next.js)   | [http://localhost:3000](http://localhost:3000)                         |
| API 서버 (NestJS)   | [http://localhost:8000](http://localhost:8000)                         |
| Swagger API 문서    | [http://localhost:8000/api/v1/docs](http://localhost:8000/api/v1/docs) |
| 화면 공유 서버 (tshare) | [http://localhost:4443](http://localhost:4443)                         |
| AI Worker 서버      | [http://localhost:9000](http://localhost:9000)                         |


> 실서버 배포 환경인 경우 localhost를 해당 서버 IP로 교체한다.

## 캡처 촬영 팁

1. **브라우저 너비**: 1920px 이상으로 설정하여 테이블 컬럼이 잘리지 않도록 한다
2. **데이터 충분히 준비**: 각 화면에 최소 3~5건의 샘플 데이터가 보이는 상태에서 캡처
3. **다양한 상태 노출**: 목록에 여러 상태(ONLINE/OFFLINE, APPROVED/PENDING 등)가 섞여 있으면 시스템 기능이 잘 보임
4. **다이얼로그 별도 캡처**: 등록/수정/삭제 다이얼로그는 목록 화면과 별도로 캡처
5. **에러/성공 피드백**: 토스트 메시지, 성공/실패 알림이 표시된 순간 캡처
6. **사이드바 포함**: 좌측 사이드바(메뉴)가 함께 보이도록 캡처하면 네비게이션 구조를 보여줄 수 있음
7. **tshare_launcher**: Electron 앱은 전체 데스크톱 화면과 함께 캡처하면 미니 모드/오버레이 위치가 잘 보임
8. **모바일 캡처**: QR 접속(#14)은 실제 모바일 기기에서 촬영하면 현장감이 높음
