# 1. KU WAVE PLAT 임베디드 서버
## 1.1 CMS(Content Management System) 서버 솔루션 제작 - 강의 스트리밍 및 레코딩 관리 
### 1.1.1  컨텐츠 통합관리 기능

-  컨텐츠 관리 서버용 웹 솔루션(CMS)은 교내 포털과 연동한 로그인 기능 제작
-  수업자동추적녹화시스템용  솔루션과  연동한  녹화기  제어(녹화시작,  일시정지, 녹화종료 등 기능 제작  
-  컨텐츠 관리 서버용 웹 솔루션(CMS)에 로그인 접속 하면(교내 포털 아이디 인증 연동 기능 포함) 녹화준비  화면이  활성화  되고  필요 정보를 입력 후 녹화제어 버튼이 활성화되면서 녹화 시작할 수 있도록 제작  
-  녹화시작 버튼을 클릭시 컨텐츠 관리 서버용 웹 솔루션(CMS)은 녹화관련 정보(FTP폴더 주소, 녹화시작,  녹화  파일이름 등)가  녹화기와  연동되도록  제작하여  녹화파일이 생성되도록 하고 녹화 종료시 FTP 서버폴더로 자동 업로드 및 파일생성 규칙에 따라 저장하도록 제작
-  컨텐츠 관리 서버용 웹 솔루션(CMS)에서  파일  업로드 상태를 모니터링할  수  있도록 제작
-  컨텐츠 관리 서버용 웹 솔루션(CMS)에 저장된 파일은 해당 컨텐츠를 제작한 강의자만 자료에 접근이 가능하도록  제작
-  컨텐츠 관리 서버용 웹 솔루션(CMS)에서 녹화 진행중인 녹화기의 중복  사용  방지를  위해  녹화기의  녹화 상태정보를 연동하여 접속 통제와 모니터링
-  관리자는  컨텐츠 관리 서버용 웹 솔루션(CMS)을  통해 녹화기  상태를  확인 및 제어 가능하도록 제작
-  기타 파일관리, 녹화관리  등 기능  관련  추가 관리설정 메뉴를  제작해야한다.  
-  키오스크  솔루션과  연동하여 녹화 작동시  상태(녹화  중 등)이  표시되도록 연동 제작
-  녹화 파일 서버 업로드 및 다운로드 기능
-  사용자가  호실별 파일 업로드  및 다운로드를 할  수  있는 기능  구현
-  교수학습센터의 온라인 교육 컨텐츠 관리․공유 솔루션 연동을 위해 파일명 규칙을 반영하여 폴더파일 생성 및 녹화파일이 전송될 수 있도록 제작


### 1.1.2 e러닝 시스템 연동
❍ 시스템 연동 기능을 제공 해야 한다

- 녹화가 완료된 시점에 자동으로 업로드 할 수 있는 기능 제공
- e러닝 폴더명 및 파일생성 규칙에 따라 파일을 생성
1. 폴더 생성 방식
      - 장비코드 > 날짜 > 강의실명 > 강의자ID_강의제목 > MP4 파일
      - ex) 001 > 20200101 > 경영관001 > 110111_강의명 > MP4 파일

    2. 파일명 생성 방식
      - 001 (장비코드 폴더) >
      - ex) 20200101_경영관001_110111_강의명. MP4

- 시스템에 파일 업로드된 상태(대기중, 업로드완료, 업로드실패)를 확인 가능 하도록 기능 제공
- 녹화된 파일에 대한 다운로드 및 미리보기 기능을 제공
- 녹화를 진행한 사용자에 대해서만 다운로드 기능 및 미리보기 기능을 제공


## 1.2 수업 공유 및 관리를 위한 솔루션 제작
### 1.2.1 수업 화면 공유 
❍ 강의실 수업화면 공유
❍ 동일 네트워크망 하에서 화면 공유 제공
 ** UI는 학교의 요구사항을 반영한다. 

### 1.2.2 강의자 화면 공유 기능
- 화면공유 송출
  - 교수자의 PC/노트북/태블릿 화면을 실시간 공유
  - 전체 화면 또는 특정 창 선택 공유
  - 웹 브라우저 기반 화면 공유 (WebRTC 활용)
- 학생 화면 수신
  - 학생들은 별도 프로그램 설치 없이 웹 브라우저로 접속
  - URL 또는 IP 주소 입력으로 즉시 
  - 실시간 화면 스트리밍 수신
- 해상도 지원
  - Full HD (1920x1080) 이상 지원
  - 자동 해상도 조정 (네트워크 상태에 따라)
- 제어 기능
  - 교수자는 화면공유 시작/중지 제어

### 1.2.3 내부 네트워크 접근
- IP 대역 기반 접근 제어
  - 특정 IP 대역에서만 화면공유 접근 허용
  - 건국대학교 내부 IP 대역 화이트리스트 등록
  - 외부 IP 접근 차단
- Wi-Fi 기반 접근 제어
  - 지정된 Wi-Fi SSID에 연결된 기기만 접근 허용
  - 강의실별 전용 Wi-Fi 또는 학교 공식 Wi-Fi만 허용
- 접근 제어 설정
  - 관리자 페이지에서 허용 IP 대역 설정
  - 허용 Wi-Fi SSID 목록 관리
  - 접근 제어 규칙 추가/수정/삭제
- 접근 시도 로그
  - 모든 접근 시도 기록 (성공/실패)
  - 비정상 접근 시도 탐지 및 알림
  - IP 주소, 접속 시간, 접속 결과 기록

### 1.2.4 상시 접근 가능한 화면 공유 서비스 
- 상시 서비스 운영
  - 화면공유 서버 24시간 상시 운영
  - 별도 방 생성 없이 호실별 고정 URL/IP 제공
  - 강의 시작 시 즉시 화면공유 가능
- 호실별 고정 주소
  - 각 강의실마다 고유한 URL 또는 IP 주소 할당
  - 예:http://classroom.konkuk.ac.kr/room301 또는 192.168.1.101
  - QR 코드 생성으로 학생들의 빠른 접속 지원
- 대기 화면
  - 교수자가 화면공유를 시작하지 않았을 때 대기 화면 표시
  - 강의실 정보, 안내 메시지 표시
  - 화면공유 시작 시 자동으로 실시간 화면으로 전환
- 자동 재연결
  - 네트워크 일시 끊김 시 자동 재연결
  - 학생 측 브라우저 자동 새로고침
- 음성 미지원
  - 화면만 공유, 음성 스트리밍은 제외
  - 강의 음성은 강의실 내 마이크/스피커 시스템 활용

## 1.3 AI 개인화 설정 및 강의지원 제작
### 1.3.1 AI 실시간 음성 명령어 인식 및 제어
- 실시간 음성 인식
  - AI Vosk 등 AI 기반 음성 인식 엔진 활용
  - 실시간 음성 스트림 처리
  - 한국어 음성 명령어 인식
- 명령어 변환
  - 음성을 텍스트로 변환 (STT)
  - 인식된 텍스트를 제어 명령어로 매핑
    * 예: "강의시작", "강의종료", "스크린 올려" 등
- 장비 제어 실행
  - 인식된 명령어에 해당하는 장비 식별
  - 프리셋에 설정된 제어 명령어 전달
  - 지정된 장비에 ON/OFF 또는 수치 조절 명령 실행
- 다양한 명령어 패턴 지원
  - 음성 명령의 다양한 표현 인식
    - 동의어 및 유사 표현 처리
  - 명령어 확인 피드백
  - 인식된 명령어를 음성 또는 화면으로 확인 메시지 출력
  - 제어 이력 기록
    - 음성 명령 시간, 인식된 명령어, 제어 내용 자동 기록
- 강의 종료 후 강의 자료 작성
- Whisper 등 AI 기반 WAV를 TXT 문서로 작성
- 강의 내용을 요약 정리
    - Ollama 등 AI 를 통한 강의 내용 요약 정리 기능 제공


### 1.3.2 AI 강의 종료 이후 강의자료 작성
- 강의 종료 후 강의 자료 작성
- Whisper 등 AI 기반 WAV를 TXT문서로 작성    
  - OpenAI Whisper, faster-whisper 등 기반 음성 인식 엔진
- 한국어 최적화 (language="ko", 키워드 힌트 적용)
- 모델 크기 선택: tiny/base/small/medium/large

### 1.3.3 AI 강의 종료 이후 강의자료 요약

  - Ollama 등 AI를 통한 강의 내용 요약 정리 기능 제공

# 2. 원격 비대면 제어 시스템(솔루션 연동)
## 2.1 원격 제어관리 웹서버 솔루션 제작
### 2.1.1 원격 제어관리
- 장비 제어 기능 제작
- 장비 목록 확인
- 강의실 리스트 개수에 제한이 없도록 기능
- 강의실 장비를 등록 할 수 있는 기능 제공
- 강의실 장비등록 개수에 제한이 없도록 기능 제공
- 강의실 선택하고 등록된 장비를 노출 되도록 기능 제공
- 강의실 장비을 선택하고 사용 가능한 명령어 기능을 제공
- 장비 사용 가능한 명령어 개수에 제한이 없도록 기능 제공
- 명령어 실행 이후 실행 결과를 알려주는 기능 제공

### 2.1.2 강의실 관리
- 강의실 정보를 관리, 수정 할 수 있는 기능을 제공해야 한다.
- 강의실 정보 관리(등록, 수정, 삭제, 상세) 할 수 있는 기능 제공
- 강의실 공간 등록에 제한이 없도록 기능 제공

### 2.1.3 알림관리
- 강의실(호실)별 알림을 등록하고 제어 할 수 있는 기능을 제공해야 한다.
- 강의실(호실)별 알림을 등록하고 전송 하는 기능 제공
- 알림 구분 (일반, 긴급, 기타)로 설정 할 수 있는 기능 제공
- 알림은 타이틀, 내용으로 구분지어 전송 할 수 있도록 기능 제공


### 2.1.4 사용자 인증
- 시스템을 사용하기 위한 사용자 인증, 관리 제어 할 수 있는 기능을 제공해야 한다.
- 사용자에 대한 관리 (등록, 수정, 삭제) 기능을 제공
- 사이트 접근에 대한 권한 기능 제공
- 사용자에 대한 검색 기능 제공

### 2.1.5 환경설정
- 시스템을 관리 및 제어 할 수 있는 기능을 제공해야 한다.
- 하드웨어 설정 기능 제공
- 하드웨어 그룹 관리 기능 제공
- 하드웨어 설정 (컨트롤러, 녹화기, 공유장비, 스트리밍 장비)을 할 수 있는 기능 제공
- 하드웨어 설정에 대한 개수 제한이 없도록 기능 제공
- 프리셋 기능(프로토콜, 통신IP, 통신포트, 프리셋명, 명령어코드) 제공


# 3. 원격 비대면 제어 시스템(솔루션 연동)
## 3.1 솔루션 제작 
### 3.1.1 서버 로그인 / 로그아웃
- 로그인 및 로그아웃 기능 제공해야 한다
- 포털 아이디 연동 및 사용자 관리(관리자 권한)
- 승인, 거부, 반려 기능 제공
- 로그인 시 상태값에 의해 로그인 기능 제한
- 사용자별 권한 메뉴만 볼 수 있어야 하며 설정이 가능 해야 함

### 3.1.2 서버 대시보드
- 대시보드 기능
- 대시보드 정보는 담당자와 협의 후 다양한 형태 구현

### 3.1.3 사용자관리
- 관리자 계정 이외 사용자를 위한 기능을 제공해야 한다.
- 사용자 추가 기능  
- 사용자 삭제 기능  
- 관리자가 사용자에게 접근 가능한 페이지 권한  
- 사용자별 채널부여 기능 제공
- 사용자별 컨텐츠별 승인 기능  

### 3.1.4 리스트 관리
- 컨텐츠를 제어하기 위한 채널 기능을 제공해야 한다.
- 채널 추가, 채널 삭제 기능  
- 채널 리스트 기능  
- 컨텐츠 제공 순서 (랜덤, 순차) 기능  
- 컨텐츠 화면 구성 (가로, 세로) 기능  
- 채널을 여러 곳에서 사용할 수 있는 구조

### 3.1.5 컨텐츠관리
❍ 채널에 포함 될 컨텐츠를 제어하기 위한 기능을 제공해야 한다.
- 컨텐츠 추가/ 삭제/ 수정 기능 
- 컨텐츠 순서 조정 기능
- 관리자가 컨텐츠 승인 기능 
- 컨텐츠 방송시간을 시작일, 종료일로 제어 할수 있는 기능 
- 컨텐츠는 동영상 및 이미지 추가 할 수 있도록 기능 
- 동영상은 순차적으로 재생되며 이미지는 재생 시간 제어
- 컨텐츠는 분할에 맞추어 등록 할 수 있도록 기능 
- 컨텐츠는 스케쥴에 의한 자동으로 동기화 되는 기능 

### 3.1.6 플레이어 관리
- 플레이어를 관리, 승인 및 관리 기능
- 플레이어명을 따로 지정 할 수 있도록 기능 
- 플레이어 IP 정보, 상태를 확인 할 수 있는 기능 
- 플레이어 승인 일자 및 승인 여부에 대한 확인 기능 
- 플레이어에 채널 부여 기능 
- 플레이어 등록 제한이 없도록 기능 
- 플레이어에 채널을 일괄 업데이트 할 수 있는 기능

## 3.2 앱제작
### 3.2.1 플레이어 앱 제작
- 설치형 플레이어로 컨텐츠 재생 기능을 제공해야 한다.
- 컨텐츠 구성에 따른 화면 재생（가로, 세로) 기능 
- 컨텐츠 제공 순서 재생 기능

- 이미지 선택적 노출 및 터치를 통한 전환 할 수 있는 기능 
- 특정 시간대 블랙아웃(컨텐츠 재생 멈춤) 기능을 
- 플레이어에 컨텐츠 제공하는 기능
- 승인, 처리 기능, IP 등록 기능 등 제작
- 강의시스템 관리 솔루션과 연동하여 강의 녹화시 상태표시
